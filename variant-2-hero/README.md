# Hero — Variant 2 (архив)

Альтернативный («второй») вариант хиро: **всё по центру** (плашка, заголовок,
лид, кнопка, мелкий текст), под кнопкой-ссылкой — таблица-лестница из
bonus-секции, под таблицей — stat-bar; над всем этим продублированы топбар и
nav (статичный) для вида «целого верха страницы».

По просьбе убран из `index.html` (на странице остался только первый вариант),
но сохранён здесь целиком, чтобы при желании вставить обратно.

## Что внутри
- `variant-2.html` — разметка: дубль топбара + nav и сама секция `#hero-v2`.
- `variant-2.css` — стили варианта 2 (+ хелпер `.nav--static`).
- `variant-2.js` — одна строка, запускающая слайдшоу фона варианта 2.
- `img-hero-v2/` — 4 фоновых фото для слайдера варианта 2 (+ README).

## Как вставить обратно
1. **CSS** — содержимое `variant-2.css` вставить в главный `<style>` хиро в
   `index.html` (перед его `</style>`).
2. **HTML** — содержимое `variant-2.html` вставить в `index.html` сразу после
   закрывающего `</section>` ПЕРВОГО хиро (у которого `id="hero"`).
3. **JS** — внутри IIFE «HERO BACKGROUND SLIDESHOW», рядом с существующей
   строкой `initHeroSlides('heroSlides', 'hero', 'img/hero/');` добавить строку
   из `variant-2.js`:
   ```js
   initHeroSlides('heroSlidesV2', 'hero-v2', 'img/hero-v2/');
   ```
4. **Картинки** — папку `img-hero-v2/` скопировать обратно в корень репозитория
   как `img/hero-v2/` (создать, если её нет).

После этого второй вариант снова появится под основным хиро.

> Зависит от общих стилей/скриптов первого хиро (`.hero--devin-v2`, `.ladder`,
> `.stat-bar`, функция `initHeroSlides`, count-up) — они остаются в `index.html`,
> отдельно копировать не нужно.
